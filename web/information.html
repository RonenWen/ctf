                          # web1

                          非常简单，直接Ctrl+U查看源代码即可

                          # web2

                          该题使用了js，禁止右键查看源代码，但是依旧可以使用Ctrl+U的方式去查看源代码

                          # web3

                          题目中提示可以抓包，那我们就抓个包看看

                          可以看到，开启了“拦截响应包”之后，我们就能在响应头里面看到Flag了

                          # web4

                          按照题目提示，访问robots.txt
                          得到以下内容：

                          
User-agent: *
Disallow: /flagishere.txt

                          随后，访问/flagishere.txt，得到flag

                          # web6

                          题目提示解压源码到当前目录，意味着当前目录下应该是有一个压缩包的
                          直接尝试index.zip,没反应，tar.gz也不行

                          # web9

                          这题是vim缓存，直接访问/index.php.swp即可

                          提示

                          （以下内容来自CSDN）
                          vim缓存泄露，在使用vim进行编辑时，会产生缓存文件，操作正常，则会删除缓存文件，如果意外退出，缓存文件保留下来，这是时可以通过缓存文件来得到原文件，以index.php来说，第一次退出后，缓存文件名为 .index.php.swp，第二次退出后，缓存文件名为.index.php.swo,第三次退出后文件名为.index.php.swn
